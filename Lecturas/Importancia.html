FASES



Análisis de requisitos: Es el proceso de levantamiento de información para lograr los requisitos solictados.

Diseño y arquitectura: Determina como funcionará incorporando la implementación tecnología considerada.

Programación: Todo se reduce en un código de programación para comenzar a construir el proyecto.

Pruebas: Consiste en comprobar que el software realice correctamente las tareas indicadas.

Documentación: Documentación del propio desarrollo del software y de la gestión del proyecto, diagramas, pruebas, manuales de usuario, manuales técnicos, etc; todo con el propósito de eventuales correcciones.

Mantenimiento: Mantener y mejorar el software para enfrentar errores descubiertos y nuevos requisitos.
